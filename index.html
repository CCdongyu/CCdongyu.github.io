Software Architecture

Posted on 2018-11-22

架构目的：架构设计的主要目的是为了解决软件系统复杂度带来的问题。
架构原则：分而治之、关注点分离。
计算密集型、存储密集型。
IO：增加缓存
数据库:读写分离，数据不一致。
高并发：负载均衡（集群化），应用无状态。
高响应：最好能异步处理。
存储集群化：入口可能成为新瓶颈。
应用集群化：需求变更快导致发布代价高，应用拆分为多个服务，通过消息中间件保证服务之间的通讯。

架构模式：分层、分割、分布式、集群、缓存、异步、冗余、自动化、安全。
关注点：性能，可用性，伸缩性，扩展性、安全
性能：分布式缓存、异步操作、使用集群、代码优化（忽略）和存储性能优化。
高可用指标。
可伸缩：一致性hash算法保证数据的一致性
可扩展性：拆分应用为不同服务。要注重版本问题。
安全：认证、授权、加密。

Effective java 3rd Edition笔记

Item 1: Consider static factory method instead of constructor

Advantage

One advantage of static factory methods is that, unlike constructors, they have names.
For example, BigInteger(int, int, Random), may return a prime. BigInteger.probablePrime(int, Random) is a better expression. It is easier to read.

A second advantage of static factory methods is that, unlike constructors, they are not required to create a new object each time they’re invoked.
For example, Boolean.valueOf(boolean), never creates an object.

A third advantage of static factory methods is that, unlike constructors, they can return an object of any subtype of their return type.

Item 18: Favor composition over inheritance

Unlike method invocation, inheritance violates encapsulation.
It is a better way to use the design pattern called Decorator pattern.

Item 19: Design and Document for inheritance or else prohibit it.

The class must document its self-use of overridable methods.
The only way to test a class designed for inheritance is to write subclasses.
Constructor must not invoke overridable methods.
The best solution to this problem i sto prohibit subclassing in classes thatare not designed and documented to be safely subclasses.

Item 20: Prefer interfaces to abstract classes

Existion classes can easily be retrofitted to implement a new interface.

Interfaces are ideal for defining mixins.

Interfaces allow for the construction of nonhierarchical type frameworks.

Item 23: Prefer class hierarchies to tagged classes.

Tagged class refers to the class that contains a tag field indication the flavor of the instance.
Tagged classes are verbose, error-prone, and inefficient, just a pallid imitation of a class hierarchy.

Item 24: Favor static member classes over nonstatic

There are four kinds of nested classes: static member classes, nonstatic member classes, anonymous classes, and local classes.

Item 50: Make defensive copies when needed.

It is essential to make a defensive copy of each mutable parameter to the constructor.

Defensive copies are made before checking the validity of parameters, and the validity ckeck is performed on the copies rather than on the originals.
