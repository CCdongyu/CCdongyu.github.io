JVM内存模型

JVM内存模型主要分为两个部分，如下图所示，方法区和堆是所有线程共有，而虚拟机栈、本地方法栈和程序计数器是线程私有的。

程序计数器

和CPU中的PC类似，Java中的程序计数器用来描述指令执行的位置，不同的是CPU中的PC存放下一条指令的地址，而Java中的程序计数器存放的是当前正在执行的字节码地址。Java程序计数器是线程私有的，Java在自己的内存中开辟了一块内存区域用来模拟CPU的PC。

虚拟机栈

线程私有。每个方法执行的时候会创建一个栈帧，用于存放局部变量表，操作数栈，动态连接以及方法出口等信息。我们平时把内存分为堆内存和栈内存，其中的栈内存指的就是虚拟机栈中的局部变量表部分。局部变量表存放了在编译期间可以知道的基本数据类型，对象引用以及返回后所指的字节码地址。局部变量所需的空间在在编译期间完成分配，进入一个方法时，栈帧的大小完全确定，运行期间不改变大小。

本地方法栈

和虚拟机栈类似，为本地方法服务。HotSpot虚拟机将本地方法栈和虚拟机栈合而为一了。

方法区

Java虚拟机规范把方法区描述为堆的一个逻辑部分，但是它却有一个别名叫做“No-Heap(非堆)”。方法区的数据是所有线程共享的。它主要用于存储被虚拟机加载的类信息、常量以及静态变量等数据。在JDK8之前，HotSpot虚拟机通常使用永久代（PermGen）实现方法区。从JDK8开始，JVM不再有Perm Generation，取而代之的是元空间（MetaSpace）。两者不同点在于，元空间不存在与JVM中，而是存在于内存地址中。（PermGen经常出现内存溢出。）

堆内存

堆是所有线程共享的内存区，几乎所有的对象实例都是在这里分配内存。堆也是JavaGC管理的主要区域。一般来说，堆可以细分为Eden、From Survivor Space、To Survivor Space以及Old Generation等。
