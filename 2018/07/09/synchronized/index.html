Synchronize In Java

Posted on 2018-07-09

Monitor

Monitor是Java中多线程同步的重要概念，是synchronized关键字实现的重要基础。

Monitor和操作系统中的Binary Semaphore（也就是Mutex）类似，线程在访问共享资源的时候，首先要获取Monitor，然后才能对共享资源进行操作。

在Java中，每个对象都和特定的Monitor关联（包括Class对象）。如果一个线程拥有某个对象的Monitor，那么在这个线程释放这个对象的Monitor之前，其他线程都无权访问这个这个对象的同步方法（这里的表述不是很严谨）。

一旦某段代码被synchronized关键字修饰，JVM就会在后台自动实现Monitor。

synchronized用法

同步方法

在需要同步的方法名前加上synchronized修饰符。在这样的情况下，同步方法在执行前会去获取当前方法所属对象的monitor，获取到后才能执行同步方法。如果monitor已被其他线程获取，则当前线程阻塞等待。在多线程的情景下，只有当monitor被释放，其他线程才能执行同步方法。

同步方法获取的monitor是当前对象的monitor。这也就意味着如果一个对象有多个同步方法时，一旦一个同步方法获取了monitor，该对象的其他同步方法均被阻塞。

同步方法只能和同步方法相同步，不会阻塞非同步方法。

同步静态方法

同步静态方法和同步方法原理类似，不同的是monitor所属的对象不同。可以知道的是，静态方法属于类，也就是说synchronized修饰的静态方法获取的是类的monitor，也就是Class对象的monitor。而同步方法获取的是当前实例的monitor。

代码块

代码块很容易理解。其实就是将对象当作锁来使用。

总结

synchronized的使用方式也就两类：修饰方法，修饰对象。而synchronized的本质就是去获取对象的monitor（在JVM中类本身对应一个Class对象）。只要清楚理解synchronized操作的对象是谁，就能理解synchronized关键字是如何同步的了。
